# SpecOS

> Structured spec in. Runnable project out.

SpecOS is a TypeScript CLI for compiling a structured spec project into a runnable application skeleton with AI.

Current MVP:

- Node.js `>= 18`
- npm-style CLI package
- project-local `spec.config.js`
- global API initialization through `spec init`
- AI compile pipeline through `spec compile <projectDir>`
- target stack: `React + Ant Design + TypeScript + Flask + MongoDB`

## Product Model

SpecOS is intentionally spec-driven:

```text
spec project -> AI compile -> runnable frontend + backend project
```

The source of truth is the spec project directory, not the generated code in `dist`.

SpecOS works best when the spec stays focused on:

- domain model
- business actions
- page intent and user-visible structure

Keep product intent in `app.spec`, and put stack/runtime details in `spec.config.js` plus a dedicated stack profile file.

## Commands

Initialize global API config:

```bash
spec init --host https://api.openai.com/v1 --auth sk-...
```

Or load it from a file:

```bash
spec init --config ./spec.global.json
```

Create a project template:

```bash
spec init --project ./examples/my-app
```

Compile a spec project:

```bash
spec compile ./examples/todo-app
```

Override output or model at compile time:

```bash
spec compile ./examples/todo-app --outDir ./dist --model gpt-4o-mini
```

Debug compile output:

```bash
spec compile ./examples/todo-app --debug
```

Projects created with `spec init --project` include a hidden `.specos/` scaffold. Runtime files such as `frontend/package.json`, `frontend/index.html`, `frontend/tsconfig.json`, `frontend/vite.config.ts`, `backend/requirements.txt`, and `.env.example` are compiled from that scaffold instead of being regenerated by the model.

Run a generated project:

```bash
spec run ./examples/todo-app
```

Install dependencies before running:

```bash
spec run ./examples/todo-app --install
```

Run in dev/watch mode:

```bash
spec run ./examples/todo-app --dev
```

## Project Config

Each spec project can define its own `spec.config.js`:

```js
export default {
  outDir: "./dist",
  stackConfig: "./stack.config.js",
  ai: {
    model: "gpt-4o-mini",
    temperature: 0.2
  },
  compile: {
    clean: false,
    verbose: true
  }
}
```

And move detailed stack settings into `stack.config.js`:

```js
export default {
  frontend: {
    framework: "react",
    frameworkVersion: "18.3.1",
    ui: "antd",
    uiVersion: "5.24.7",
    language: "typescript",
    languageVersion: "5.8.2",
    nodeVersion: "20",
    host: "0.0.0.0",
    port: 3000,
    apiBasePath: "/api",
    proxyTarget: "http://127.0.0.1:5000"
  },
  backend: {
    framework: "flask",
    frameworkVersion: "3.0",
    language: "python",
    languageVersion: "3.11",
    host: "127.0.0.1",
    port: 5000,
    entry: "app.py"
  },
  data: {
    engine: "mongodb",
    engineVersion: "7",
    uri: "mongodb://127.0.0.1:27017/specos"
  }
}
```

Config precedence:

```text
CLI flags > project spec.config.js > global spec init config
```

`auth` should stay in the global config, not in project files.

The stack profile is the right place for:

- framework and library names
- exact package versions
- frontend/backend host and port
- API base path and frontend proxy target
- database URI and database name
- backend entrypoint and CORS allowlist

## Spec Shape

Recommended top-level blocks:

- `App`: one-line product description
- `Goal`: what the system is meant to do
- `Entity`: domain objects and field constraints
- `Action`: business capability, input, result, and rules
- `Page`: page route plus a direct, readable page description
- `Component`: reusable modal or form content
- `State`: page-level data source bindings

Recommended page sections:

- `Summary`
- `Query`
- `Load`
- `Header`
- `Filters`
- `Content`
- `Empty`
- `Footer`

Write pages as a readable page brief, not as React code or a UI component tree.

Prefer:

- `Search -> reload todos`
- `Add Todo -> open CreateTodoModal`
- `Complete -> CompleteTodo(id = row.id) -> reload todos`

Avoid:

- framework names in `app.spec`
- API URLs as the main product definition
- CSS or layout implementation details
- hook, dispatch, or component-library specific wording

## Example Project

```text
examples/todo-app/
├── spec.config.js
├── stack.config.js
└── app.spec
```

The MVP scans `.spec` files recursively and sends the merged spec context to the model.

Example:

```spec
App: Todo Manager

Goal:
  Manage personal todos with a clear list page and a quick create flow.

---

Entity Todo:
  id: string (primary)
  title: string (required, maxLength=100)
  completed: boolean (default=false)

---

Action CreateTodo:
  Input:
    title
  Do:
    insert Todo:
      title = input.title
      completed = false
  Return:
    success
    id

---

Action SearchTodos:
  Input:
    searchKey = "" (optional)
    page
    pageSize
  Do:
    query Todo:
      where completed = false
      and title contains searchKey
    paginate by page, pageSize
  Return:
    data: Todo[]
    total

---

Page TodoPage (/todos):
  Summary:
    List open todos, search them, and create new ones.

  Query:
    searchKey = ""
    page = 1
    pageSize = 10

  Load:
    todos = SearchTodos(searchKey, page, pageSize)

  Header:
    text("Todo Manager", align=center)
    button("Add Todo", primary):
      onClick:
        openModal CreateTodoModal

  Filters:
    input(searchKey)
    button("Search"):
      onClick:
        dispatch SearchTodos
        refresh todos

  Content:
    table(todos):
      columns:
        id
        title
        action:
          button("Complete"):
            onClick:
              dispatch CompleteTodo(id = row.id)
              refresh todos

---

Component CreateTodoModal:
  modal("Create Todo"):
    form:
      field title (input, required)
    onSubmit:
      dispatch CreateTodo(title = form.title)
      closeModal
      refresh todos

---

State:
  todos:
    source: SearchTodos
```

## Compile Output

Generated files are written into the configured `outDir`.

SpecOS also writes metadata into:

```text
dist/.specos/
├── compile-log.json
├── file-manifest.json
└── prompt-trace.json
```

## Local Development

Install dependencies and build:

```bash
npm install
npm run build
```

Run the compiled CLI locally:

```bash
node ./lib/index.js compile ./examples/todo-app
```
